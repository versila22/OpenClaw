## GitHub auth dans le conteneur Docker
- Problème : pas de `gh` CLI ni de credential helper dans le conteneur → impossible de push
- Le token `GITHUB_TOKEN=ghp_...` dans .env était invalide (401)
- Solution : fichier `/root/.config/gh/hosts.yml` copié dans `/opt/openclaw/config/.gh/hosts.yml`
  - Monté automatiquement via le volume `./config:/data/.openclaw`
  - Visible dans le conteneur à `/data/.openclaw/.gh/hosts.yml`
  - chmod 644 nécessaire (conteneur tourne en user `node`)
- Script credential helper créé : `/data/.openclaw/workspace/.local/bin/git-credential-gh-hosts.sh`
  - Lit le oauth_token depuis hosts.yml
  - Configuré en global git credential.helper
  - Tous les repos du workspace peuvent maintenant push
