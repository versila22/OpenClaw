
## 2026-03-23 : Projet YouTube Uploader & Incident de Sécurité

**Projet : `openclaw-youtube-uploader` (Pipeline Vidéo Automatisé)**
*   **Architecture :** Dépôt Git indépendant créé et poussé sur GitHub pour isoler le code de génération et d'upload YouTube du projet OpenClaw principal.
*   **Environnement :** Environnement virtuel Python `youtube_env`.
*   **Composants développés et validés ("MVP") :**
    *   **TTS :** `test_tts.py` implémenté avec l'API ElevenLabs (via la variable d'environnement `ELEVENLABS_API_KEY`). Configuration sur la voix "Rachel" (`21m00Tcm4TlvDq8ikWAM`) et le modèle `eleven_multilingual_v2`.
    *   **Composition :** `compose_video.py` opérationnel. Downgrade volontaire vers `moviepy<2.0.0` (v1.0.3) pour contourner les breaking changes de la v2. Génère avec succès un fond noir (1080p) synchronisé sur la durée de l'audio.
    *   **Upload YouTube :** `upload_youtube.py` implémenté avec le flux OAuth2 OOB (Out-Of-Band) `run_console()` pour le serveur headless. Utilise un client OAuth "Desktop App".
    *   **Débogage OAuth :** L'erreur 403 `access_denied` de l'écran de consentement a été patchée en ajoutant manuellement `jerome.jacq@gmail.com` comme "Utilisateur test" dans la console GCP (Projet: OpenClaw-YouTube-Uploader).

**Post-Mortem Sécurité (Incident Clos) :**
*   **Alerte :** Fuite accidentelle d'une clé API Google Cloud (Gemini, Projet: PUAProject) due à l'inclusion erronée du fichier `config/mcporter.json` lors de l'initialisation du dépôt `openclaw-youtube-uploader`.
*   **Remédiation (Succès) :**
    1.  Révocation immédiate de la clé sur Google Cloud.
    2.  Éradication totale du fichier de l'historique Git via `git filter-repo --path config/mcporter.json --invert-paths --force` suivie d'un `git push --force`.
    3.  Création d'une nouvelle clé API restreinte stricte (uniquement "Generative Language API") et réinjection en local hors du dépôt.
